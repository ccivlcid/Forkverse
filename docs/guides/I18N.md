# I18N.md — Internationalisation Guide

> **Source of truth** for language support, locale configuration, and i18n implementation.

---

## 1. Supported Languages

Forkverse supports four languages:

| Code | Language | Native Name | Direction |
|------|----------|-------------|-----------|
| `en` | English | English | LTR |
| `ko` | Korean | 한국어 | LTR |
| `zh` | Chinese (Simplified) | 中文 | LTR |
| `ja` | Japanese | 日本語 | LTR |

---

## 2. Two Language Dimensions

Forkverse has two independent language settings per user:

| Setting | DB Column | Purpose | Affects |
|---------|-----------|---------|---------|
| **UI Language** | `users.ui_lang` | Interface display language | Nav labels, placeholders, error messages, date formats |
| **Default Post Language** | `users.default_post_lang` | Language assumed when composing | LLM `--lang` flag, post metadata |

These are independent. A Korean user may write posts in English, or an English user may write posts in Japanese.

---

## 3. UI Language

### Values
`en` | `ko` | `zh` | `ja` (default: `en`)

### Where it applies
- Sidebar navigation labels
- Composer placeholder text
- Error/empty state messages
- Toast notification text
- Date/time formatting (locale-aware `Intl.DateTimeFormat`)
- Settings page labels

### Implementation

```typescript
// packages/client/src/stores/uiStore.ts
interface UiState {
  lang: 'en' | 'ko' | 'zh' | 'ja';
  t: (key: string) => string;
  setLang: (lang: UiState['lang']) => void;
}
```

Translation strings live in `packages/client/src/i18n/{lang}.ts`:
```
packages/client/src/i18n/
├── en.ts    ← default
├── ko.ts
├── zh.ts
└── ja.ts
```

The `t()` function looks up strings by key:
```typescript
t('feed.empty.title')  // → "No posts yet." / "아직 게시물이 없습니다." / etc.
```

### Change flow
1. User selects language in Settings
2. Client calls `PUT /api/auth/me` with `{ uiLang: 'ko' }`
3. Server updates `users.ui_lang`
4. Client `uiStore.setLang('ko')` triggers re-render

## 4. Post Language

### Values
`auto` | `en` | `ko` | `zh` | `ja` (default: `auto`)

### Where it applies
- Passed as `--lang` flag in LLM system prompt
- Stored in `posts.lang` column
- Displayed as a `--lang=ko` badge on the post card
- Used for feed filtering (future)

### Auto-detection

When `default_post_lang` is `auto` (or the composer lang selector is set to auto), the server runs a lightweight language detection before calling the LLM:

```typescript
// packages/server/src/lib/lang-detect.ts
// Uses Intl.Segmenter + Unicode range heuristics
export function detectLang(text: string): 'en' | 'ko' | 'zh' | 'ja' {
  const koRange = /[\uAC00-\uD7A3]/;   // Hangul syllables
  const jaRange = /[\u3040-\u30FF]/;   // Hiragana + Katakana
  const zhRange = /[\u4E00-\u9FFF]/;   // CJK Unified Ideographs

  if (koRange.test(text)) return 'ko';
  if (jaRange.test(text)) return 'ja';
  if (zhRange.test(text)) return 'zh';
  return 'en';
}
```

Detection priority: Korean → Japanese → Chinese → English (default).

### Composer UI

The composer shows a language selector next to the model selector:

```
┌─ Composer Bar ──────────────────────────────────────────────────────────┐
│ > Write in any language. LLM translates to CLI.                         │
│ ┌──────────────────────────────────────────────────────────────────┐    │
│ │ Type your post here...                                      |    │    │
│ └──────────────────────────────────────────────────────────────────┘    │
│ Cmd+Enter · [claude-sonnet ▾] · [lang: auto ▾]  [📎 repo]              │
└──────────────────────────────────────────────────────────────────────────┘
```

The dropdown shows:

## 5. LLM Prompt Localisation

The system prompt in `packages/llm/prompts/system.md` is in English and instructs the LLM to use the `--lang` flag from the user's message. The LLM handles multilingual input natively.

The `--lang` flag tells the LLM:
- What language code to set in the CLI output
- To preserve the original message text in the user's language (not translate it)

```
User writes: "오늘 드디어 사이드 프로젝트를 완성했어요!"
LLM detects: ko
LLM outputs: terminal.social post --user jiyeon_dev --lang ko --message "오늘 드디어 사이드 프로젝트를 완성했어요!" --tags sideproject,achievement --visibility public
```

## 6. Database Schema

```sql
-- 005_add_language_columns.sql
ALTER TABLE users ADD COLUMN ui_lang TEXT NOT NULL DEFAULT 'en';
ALTER TABLE users ADD COLUMN default_post_lang TEXT NOT NULL DEFAULT 'auto';
```

| Column | Values | Default |
|--------|--------|---------|
| `ui_lang` | `en`, `ko`, `zh`, `ja` | `en` |
| `default_post_lang` | `auto`, `en`, `ko`, `zh`, `ja` | `auto` |

---

## 7. Settings Screen

Both settings are in `/settings` under the `// language` section:

```
$ set --ui-lang=[en ▾]
  English · 한국어 · 中文 · 日本語

$ set --default-post-lang=[auto ▾]
  auto-detect · en · ko · zh · ja
```

Changes take effect immediately (optimistic update) and are persisted via `PUT /api/auth/me`.

---

## 8. Conventions

- **Never hardcode display strings** in components — always use `t()` from `uiStore`
- **Translation files** live in `packages/client/src/i18n/`; keep keys consistent across all four files
- **Date formatting** uses `Intl.RelativeTimeFormat` with the active `ui_lang` locale
- **Post language** is always stored as an ISO 639-1 two-letter code (`en`, `ko`, `zh`, `ja`)
- **Auto-detect** runs server-side before LLM call, never client-side
- **RTL not supported** — all four languages are LTR; no RTL layout needed
